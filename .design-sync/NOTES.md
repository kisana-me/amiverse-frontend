# design-sync notes — Amiverse

Repo-specific gotchas for future syncs. Run everything from `frontend/`.

## Shape

- This is a **Next.js 16 app**, not a published design-system package: there is no
  `dist/`, no Storybook, and no `.d.ts` tree. The converter runs in **synth-entry
  mode** (`[NO_DIST]` is expected, not a failure) and builds the bundle straight
  from `src/`.
- `node_modules/Amiverse` is a **self-symlink to the package root** (`ln -sfn ..
  Amiverse`), created so the converter can resolve `PKG_DIR` and so authored
  previews can `import { X } from 'Amiverse'`. It is not in git — **recreate it on
  a fresh clone**:
  `ln -sfn .. node_modules/Amiverse`
- `.design-sync/node_modules` is a symlink to `.ds-sync/node_modules` so the
  committed `overrides/source-kit.mjs` fork can resolve `ts-morph`. Also
  per-clone: `ln -sfn ../.ds-sync/node_modules .design-sync/node_modules`

## Build order (re-sync)

```sh
ln -sfn .. node_modules/Amiverse                              # once per clone
ln -sfn ../.ds-sync/node_modules .design-sync/node_modules    # once per clone
node .design-sync/build-css.mjs                               # ALWAYS before the converter
node .ds-sync/resync.mjs --config .design-sync/config.json --node-modules ./node_modules \
  --out ./ds-bundle --remote .design-sync/.cache/remote-sync.json
```

`build-css.mjs` is **not optional**: `cfg.cssEntry` points at
`.design-sync/.cache/ds-styles.css`, which is gitignored and regenerated by that
script. Skipping it fails the build with `! cssEntry: … not found`.

## The `source-kit.mjs` fork (`cfg.libOverrides`)

Three deviations, all forced by the app shape:

1. **`src/app/` is excluded** from src discovery. Route entries pull the Next
   server runtime (`@opentelemetry/api`) and `globals.css` (`@import
   "tailwindcss"`, unresolvable by esbuild) into the synth entry, and the build
   fails outright without this.
2. **Default exports are re-exported onto the global.** Nearly every component
   here is `export default function …`, which `export * from` does NOT re-export
   — without this only ~24 of 71 components reached `window.Amiverse`.
3. **Export names come from the file path, not the declared identifier.** Several
   components carry a copy-pasted identifier: `overlay/Overlay.tsx` declares
   `MainHeader`, `post/AccountIcon.tsx` declares `Account`, `post/YouTube.tsx`
   declares `Content`, `post/ListedHeader.tsx` declares `Header`. Path naming
   leaves exactly one true collision — `components/header/Header.tsx` vs
   `features/post/components/Header.tsx` — resolved by qualifying the later path
   with its feature dir: **`PostHeader`**. If a new collision appears, the build
   prints `name collision: X → Y`; check it is the qualification you want.
4. `'features'` is treated as a generic container dir so feature slices group by
   slice name (`account`, `feed`, `form`, …), not by `features`.

## Two repo-owned modules the bundle depends on

- **`.design-sync/process-shim.mjs`** — imported FIRST by both the synthesized
  entry and `design-runtime.tsx`. The Next client runtime reads `process.env.*`
  at module scope; without the shim every card dies with
  `ReferenceError: process is not defined` and 0/71 render. If you ever see that
  error again, the shim lost its first-import position.
- **`.design-sync/design-runtime.tsx`** — merged via `cfg.extraEntries`, exports
  `DesignRoot` and is `cfg.provider`. It supplies a stub Next app-router context
  (`useRouter` otherwise throws `invariant expected app router to be mounted`)
  plus `UIProvider → OverlayProvider → ToastProvider → CurrentAccountProvider →
  PostsProvider → EmojiProvider`. Each provider in that chain was added because
  a component demanded it; removing one re-breaks specific cards
  (`CurrentAccountProvider` → ActionPrompt/Drawings/Media/SensitiveGate,
  `PostsProvider` → Reactions/ConsoleReaction, `EmojiProvider` →
  Console/Reactions/ListedPost/FeaturedPost).

## Static assets are NOT in the bundle

Components reference app-served absolute paths, which 404 outside the app:
`/static-assets/images/amiverse-logo-alpha-400.png` (`MainHeader`, `Header`,
`InitialLoading` — hard-coded, not overridable) and `/ast-imgs/icon.png` /
`banner.png` (every avatar/banner fallback). Previews avoid this by always
passing explicit image URLs; `MainHeader` cannot and shows a broken logo. This
is documented for the design agent in `conventions.md`.

## Styling

- **Tokens live in JS, not CSS.** `src/providers/UIProvider.tsx` injects the
  `--*` custom properties from a `<style>` element. `.design-sync/tokens.css`
  **mirrors those values by hand** so designs are styled without the provider —
  see Re-sync risks.
- `--content-color` is deliberately **not** shipped (retired from the DS). It is
  still referenced by `src/app/**` pages, which are outside the synced surface.
- Five custom properties are referenced by shipped components but never defined
  by UIProvider: `--text-primary`, `--text-secondary`, `--bg-secondary`,
  `--bg-color`, `--main-container-background-color`. With the user's agreement
  these are **aliased onto the real tokens** in `tokens.css`. The app itself
  still leaves them undefined — that is an app-side bug worth fixing at source.
- **Tailwind v4 is part of the idiom**, mixed with CSS Modules. There is no
  compiled Tailwind stylesheet in the repo, so `build-css.mjs` compiles one from
  the classes used under `src/` and concatenates the tokens.

## Excluded from the sync

- `src/components/post/form.tsx` (`Form`) — the deprecated post form, no longer
  imported anywhere. Superseded by `features/form/components/PostForm.tsx`.
- Seven data providers (`AccountsProvider`, `CurrentAccountProvider`,
  `EmojiProvider`, `FeedsProvider`, `NotificationsProvider`, `PostsProvider`,
  `TrendsProvider`) — they fetch from the API on mount and mean nothing to a
  design. They stay in the bundle as exports; only their cards are excluded.
  `UIProvider`, `OverlayProvider` and `ToastProvider` are kept: they are the
  preview provider chain (`cfg.provider`) and pure client state.

## Environment

- Playwright's chromium needs system libraries that are NOT installed by
  `playwright install chromium` alone. On this WSL2 box:
  `sudo npx playwright install-deps chromium` (needs an interactive sudo
  password — an agent cannot run it).
- The box ships **no CJK font**. Install `fonts-noto-cjk` before the render
  check, or every Japanese string in the screenshots is tofu and grading is
  worthless.

## Known render warns

The 2026-07-24 sync ended with **zero** warn lines (71/71 clean). Anything that
appears on a future run is new — look at it before recording it here. Two were
triaged away by reworking the preview rather than accepting them:

- `Modal` tripped `[GRID_OVERFLOW]` (a `<dialog>` in the top layer escapes its
  grid cell). Fixed with `cfg.overrides.Modal = {cardMode:'single',
  primaryStory:'Default'}` — keep that override.
- `ActionPrompt` and `InfiniteScrollSentinel` tripped `[RENDER_THIN]` /
  identical-variants because their second export rendered nothing visible. Both
  previews were cut down to the cells that actually render.

## Preview authoring facts learned the hard way

- `post.visibility` must be `opened` (not `public`) or FeaturedInfo renders
  「公開状態不明」.
- `EmojiType.name` carries the **emoji character itself** when `image_url` is
  empty — `EmojiGrid` and `Reactions` render `name` verbatim, so English names
  produce a jumbled text grid.
- `AccountBanner` with no `banner_url` falls back to a 404 asset; never author a
  "no banner" variant.

## Re-sync risks

- **`.design-sync/tokens.css` is a hand-kept mirror of `UIProvider.tsx`.** Nothing
  detects drift. Whenever the theme block in UIProvider changes, update
  `tokens.css` in the same commit. The alias block at the bottom has no upstream
  at all — it exists only here.
- **The `source-kit.mjs` fork tracks upstream.** On re-sync, diff it against
  `.ds-sync/lib/source-kit.mjs` and merge upstream changes; the four deviations
  above are the only intentional differences.
- **`.design-sync/previews/_fixtures.ts` types against `@/types/*`.** If
  `PostType` / `AccountType` change shape, previews may compile but render
  wrong. Sample images are inline SVG data URIs on purpose — never point them at
  a network URL.
- Component **names are derived from file paths**. Renaming or moving a component
  file renames its card and orphans its grade; the diff will show it as a
  delete + add.
- `.ds-sync/` is staged from the bundled skill — re-copy it every sync, never
  reuse a stale copy.
- **`design-runtime.tsx` mirrors `src/app/layout.tsx`'s provider chain by hand.**
  If a new provider is added to the layout, components that read it will render
  blank in designs until it is added here too.
- **The Tailwind build is scoped to classes used under `src/`.** A component that
  starts using a new utility gets it automatically on the next
  `build-css.mjs` run — but a design agent writing arbitrary utilities does not.
  `conventions.md` tells the agent to prefer `var(--…)` inline styles; keep that
  advice true if the CSS strategy changes.
- Only ~40 of 71 components have authored previews. The rest ship the floor card
  or a bare default render and are the standing offer for incremental authoring
  on any re-sync — `.design-sync/previews/` carries forward.
